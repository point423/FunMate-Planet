# AI 功能说明文档

## 功能概述

FunMate Planet 集成了基于本地大语言模型的 AI 智能助手，为用户提供个性化的活动推荐和搭子匹配建议。

## 技术架构

### 使用的模型
- **模型名称**: Qwen2.5:1.5B
- **模型提供商**: Ollama（本地部署）
- **模型大小**: 约 986 MB
- **特点**: 轻量级中文模型，适合本地部署，支持中文对话和理解

### 服务架构
前端 (Vue 3) → Nginx (反向代理) → Spring Boot 后端 → OpenAI Java SDK → Ollama API → Qwen2.5 模型

## 实现的功能

### 1. AI 活动建议助手
- **接口**: `POST /api/ai/suggest`
- **功能**: 根据用户的兴趣标签、地理位置和自然语言查询，生成个性化的活动推荐
- **输入参数**:
    - `tags`: 用户兴趣标签（如：运动、摄影、音乐）
    - `location`: 用户位置信息
    - `query`: 用户的自然语言问题（如："推荐一些户外活动"）
- **输出**: 结构化的活动建议和搭子匹配推荐

### 2. 智能对话能力
- 支持中文自然语言理解
- 上下文相关的个性化推荐
- 结合用户画像（标签、位置）提供定制化建议

## 技术实现细节

### 后端实现
- **框架**: Spring Boot 3.4.3 + Java 21
- **SDK**: OpenAI Java SDK 2.8.0（兼容 Ollama API）
- **配置**:
    - `ai.api-key`: ollama（本地无需真实密钥）
    - `ai.base-url`: http://ollama:11434/v1
    - `ai.model`: qwen2.5:1.5b
- **超时设置**: 10 分钟（适应本地模型推理速度）

### 前端实现
- **框架**: Vue 3 + TypeScript
- **HTTP 客户端**: Axios（超时设置为 300 秒）
- **UI 组件**: 聊天界面，支持消息历史和加载状态显示

### 部署架构
- **容器化**: Docker Compose 编排所有服务
- **Ollama 服务**: 独立容器，端口映射 11434
- **网络配置**: 后端通过 Docker 内部网络访问 Ollama

## 遇到的挑战与解决方案

### 1. 模型响应速度慢
**问题**: 在虚拟机环境中，Ollama 推理需要 300-600 秒

**解决方案**:
- 增加 Nginx 代理超时时间至 600 秒
- 增加 Axios 请求超时至 600 秒
- 增加 OpenAI SDK 超时至 5 分钟
- 在前端 UI 中添加加载状态提示

### 2. JWT 认证问题
**问题**: 浏览器请求返回 401，但 curl 测试正常

**原因**: JwtUtil 使用硬编码密钥，与配置文件不一致

**解决方案**: 修改 JwtUtil 使用 `@Value` 注解读取配置，统一密钥为 `FunMatePlanetSecretKeyForJwtTokenGeneration2024`

### 3. Ollama 服务连接
**问题**: 后端容器无法访问 Ollama 服务

**解决方案**:
- 配置 Ollama 监听所有接口 (`OLLAMA_HOST=0.0.0.0`)
- 使用 Docker 服务名进行内部通信

### 4. 磁盘空间不足
**问题**: Docker 镜像下载时根分区已满（95%）

**解决方案**: 清理 Docker 缓存释放 13.61GB 空间

## 性能指标

- **模型加载时间**: ~2.3 秒
- **首次推理时间**: ~300-600 秒（受硬件限制）
- **后续推理时间**: ~0.6-2 秒（模型已缓存）
- **内存占用**: 约 1GB（模型 + 运行时）
