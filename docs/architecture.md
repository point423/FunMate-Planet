# 趣搭星球 - 系统架构设计

## 一、技术选型确认
| 层级 | 选择 | 理由 |
|------|------|------|
| 前端框架 | Vue 3 + TypeScript + Vite | 采用组合式 API 实现业务逻辑高度解耦，Vite 提供了极速的热更新开发体验。 |
| 后端框架 | Spring Boot 3.5.11 | 企业级标准框架，内嵌 Tomcat 容器，支持微服务化扩展，自动配置极大提升开发效率。 |
| 持久层框架 | Spring Data JPA | 强大的 ORM 映射能力，简化了用户、活动、日记等复杂实体间的关联操作。 |
| 数据库 | MySQL 8.0 | 成熟稳定的关系型数据库，通过外键约束确保社交关系及活动记录的数据一致性。 |
| 缓存/Geo | Redis | 高性能内存数据库，利用其 Geo 空间索引实现毫秒级的“附近的人”物理距离计算。 |
| 部署方式 | Docker + Docker Compose | 实现环境一致性（DevOps），通过容器化技术一键拉起后端服务与中间件环境。 |

## 四、系统交互流程 (核心闭环)

### 4.1 附近的人发现流程
```text
用户端 (前端)             后端服务 (Spring Boot)           存储 (Redis/DB)
  │                         │                               │
  ├── 1. 上报经纬度 ───────►│                               │
  │                         ├── 2. 写入/更新位置数据 ──────►│ (Redis Geo)
  │                         │                               │
  ├── 3. 请求附近的人 ─────►│                               │
  │                         ├── 4. 搜索半径内用户 ID ──────►│ (Redis Geo)
  │                         │                               │
  │                         ├── 5. 根据 ID 查用户信息 ─────►│ (MySQL)
  │◄── 6. 返回用户列表 ─────┤                               │
```

### 4.2 活动从发起到评价流程
```text
 发起人 (User A)           后端服务 (ActService)            参与者 (User B)
  │                         │                               │
  ├── 1. 创建活动 ─────────►│                               │
  │                         ├── 2. 推送/展示活动 ──────────►│
  │                         │                               │
  │                         │◄── 3. 申请加入活动 ──────────┤
  │                         │                               │
  ├── 4. 确认结束活动 ─────►│                               │
  │                         ├── 5. 触发评价/日记引导 ──────►│
  │                         │                               │
  ├── 6. 对 B 进行评价 ────►│                               │
  │                         │◄── 7. 对 A 进行评价 ──────────┤
  │                         │                               │
  │                         ├── 8. 计算得分并归档日记 ─────►│ (MySQL)
```

## 五、核心业务逻辑说明
1. **附近的人**：通过 Redis Geo 空间索引，根据经纬度计算用户间物理距离并排序。
2. **综合评价**：活动结束后，参与者互投“高/中/低”。系统将其转换为分值（如 3/2/1），计算平均分更新至用户画像。
3. **活动日记**：活动结束后自动关联参与者，支持多图上传和标签聚合。
