- **后端框架**：Spring Boot 3.5.11 + Spring Data JPA。理由：生态成熟，JPA 简化了数据库 CRUD 操作，开发效率高。

## 遇到的问题和解决
1. **问题**：最开始建了一个pom.xml在最外层，为后面做准备。后来又在backend文件夹里建了一个pom.xml，两个混在一起导致maven依赖出错。
   **解决**：删除最外层pom.xml。
2. **问题**：不同电脑在项目运行时要求相应版本的sql,redis太麻烦。
   **解决**：利用 Docker部署。
3. **问题**：Docker 镜像拉取超时：受限于网络环境，直接从 Docker Hub 拉取镜像频繁超时。
   **解决**：配置国内镜像加速器，如 DaoCloud、以及手动执行 docker tag 重命名镜像
4. **问题**：基础镜像体积过大：标准的 openjdk 镜像下载极慢。
   **解决**：更换为极简的 liberica-openjdk-alpine (约 30MB) .
