### 2. 技术选型

- 前端框架：`Vue 3 + TypeScript + Vite`
  理由：Vue 3 组件化能力强，适合本项目多页面、多组件的组织方式；TypeScript 有助于提升代码规范性与可维护性；Vite 启动和构建速度快，适合课程项目的快速迭代开发。

- 路由管理：`Vue Router`
  理由：能够清晰管理多页面跳转关系，并支持登录拦截、嵌套路由和页面权限控制，适合本项目的页面结构设计。

- 状态管理：`Pinia`
  理由：便于统一管理用户登录状态、个人信息和部分页面共享数据，使前端架构更清晰，减少组件间传值复杂度。

- UI 组件与样式方案：`Element Plus + 自定义 CSS 变量`
  理由：Element Plus 可以提高基础交互组件开发效率；自定义 CSS 变量便于统一颜色、字体、间距和组件样式，满足项目整体视觉设计需要。

## 遇到的问题和解决

1. 问题：项目初期前端页面较多，如果没有统一的目录结构和组件划分，后续开发容易混乱。

   解决：在架构设计阶段先划分页面层、组件层、状态层、接口层和工具层，明确各目录职责，保证后续实现时结构统一。

2. 问题：页面功能之间存在较强关联，例如登录、标签选择、首页跳转、聊天和活动模块之间需要清晰的页面流转关系。

   解决：提前梳理核心交互流程，设计统一的路由结构和页面跳转逻辑，避免后期频繁返工。

3. 问题：前端既要满足课程项目开发效率，又要兼顾页面效果和后续维护性，技术选型需要在易用性和规范性之间平衡。

   解决：选择 Vue 3 + TypeScript + Vite 作为核心技术栈，兼顾开发效率、组件化组织能力和代码可维护性。

4. 问题：UI 设计方案如果没有沉淀为统一规范，后续各页面容易出现风格不一致的问题。

   解决：在架构设计阶段同步整理配色、字体、通用按钮和标签样式等前端设计规范，为页面实现提供统一依据。

## 心得体会

通过本次软件架构设计，我更加理解了前端工作不仅仅是“把页面做出来”，更重要的是在开发前先把页面结构、模块划分、技术方案和交互流程设计清楚。只有前期架构明确，后续实现阶段才能更顺畅，也更有利于团队协作。

这次工作也让我认识到，前端架构设计和 UI 设计、用户流程设计是紧密相关的。一个好的前端架构不仅要方便开发，还要能够支撑完整的用户体验流程。今后在做项目时，我会更加重视前期的结构规划与文档整理，而不是一开始就直接进入编码实现。
