## 遇到的困难与解决

### 困难 1: Ollama 模型响应超时
**问题描述**: 在虚拟机环境中，Ollama 推理需要 300-600 秒，导致前端请求超时

**解决过程**:
1. 首先发现是 Nginx 默认 60 秒超时，修改为 600 秒
2. 然后发现 Axios 默认 10 秒超时，修改为 600 秒
3. 最后增加 OpenAI SDK 的超时配置至 10 分钟
4. 通过多层超时配置确保请求能完整执行

**收获**: 理解了分布式系统中超时配置的重要性，学会了逐层排查问题的方法

### 困难 2: JWT 认证失败（401 错误）
**问题描述**: 浏览器请求一直返回 401，但 curl 测试正常

**解决过程**:
1. 检查前端是否正确发送 Authorization header（确认已发送）
2. 检查后端日志，发现没有 JWT 验证失败的异常
3. 深入分析 JwtUtil 代码，发现使用硬编码密钥
4. 修改为从配置文件读取，统一密钥配置
5. 强制重建 Docker 容器使修改生效

**收获**: 掌握了 JWT 认证机制，学会了使用 `@Value` 注解读取配置，理解了 Docker 容器重建的必要性

### 困难 3: Docker 环境问题
**问题描述**:
- 磁盘空间不足导致镜像下载失败
- 容器名称冲突
- Ollama 模型丢失

**解决过程**:
1. 使用 `docker system prune` 清理缓存释放空间
2. 使用国内镜像源（docker.m.daocloud.io）加速下载
3. 删除冲突容器后重新启动
4. 重新下载 Ollama 模型并确保卷挂载正确

**收获**: 熟悉了 Docker 常见问题排查，掌握了容器管理和数据持久化的最佳实践

### 困难 4: 前后端联调问题
**问题描述**: Discover 接口缺少必需的位置参数，导致 500 错误刷屏

**解决过程**:
1. 分析后端日志，定位到 `MissingServletRequestParameterException`
2. 修改 `@RequestParam` 为 `required = false`
3. 添加空值检查，返回空列表而不是报错
4. 前端添加位置信息的 localStorage 存储逻辑

**收获**: 学会了如何设计健壮的 API 接口，理解了前后端参数校验的重要性
